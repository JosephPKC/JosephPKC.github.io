Riptide

Feb 2017 - Mar 2017

School, Machine Learning, Completed

Github Link

What is Riptide?

    Riptide is a machine learning project for a course on data mining/machine learnining in UCI (COMP SCI 178), with the ultimate goal of placing as high as possible in a class competition on Kaggle. Riptide consists of three things: a neural network, a gradient-boosted random forest, and an ensemble of the two.

Development

    Riptide was made in Python 2.7, with a basic learner provided by the instructor. My team and I utilized various different libraries, including keras, numpy, matplotlib, scikit-learn, and collections to help build and train the learners.


    We messed with the parameters of the learners constantly, changing it up, re-training, testing and evaluating, and recording the scores. We tried to squeeze out every bit of performance it can give us. This constant re-training led to long hours of waiting for the learners to train and test, just so we can see if our slight changes made any improvements. On top of that, while reconfiguring the parameters, we also had to make sure our model wasn't too simple nor too complex.


    The neural network was created and trained with keras, and evaluated with scikit-learn. The network was trained on a set of rainfall data. It began with 1 hidden layer, a few nodes, and no other optimizations. Compared to the random forests, the neural network performed quite poorly, which we attributed to overfitting (model is too complex, so the learner could not make accurate general predictions). We later tried more complex models- more hidden layers, having the number of nodes equal to the square root of the product of input and output nodes, and implementing dropout. After trying out several more complex models, we came to the conclusion that our network was actually underfitting (model was too simple, so the learner could not accurately predict anything). We implemented a few more optimizations, like having uniformly random input numbers, and having our activation function be the sigmoid function. We also configured the network to try to minimize the binary cross-entropy loss. We ended up with a decent model with the training and test accuracies fairly close to each other, and a fairly high ROC curve (at least compared to the other learners from other teams).


    The random forest was made from the base learner provided to us by the instructor. We boosted each tree in the random forest via gradient boosting, and lowered the number of features for the trees. Almost immediately, the random forests performed spectacularly, especially compared to the neural network. We still constantly re-trained the random forest- changing the maximum height allowed, minimum number of leaves allowed, and the number of trees in the forest. Because of its performance, there wasn't much to try to improve for the random forest, so we spent most of our time on the network.


    The ensemble of learners was built from the neural network and the random forest. The ensemble would have both learners train on the data, and predict on some input data. It would then choose whatever the majority prediction from the two learners was as its prediction. This was the simplest part of the three, as most of the hard work was already finished.


Evaluation and Conclusion

    The project required us to submit our learners onto Kaggle for placement. Kaggle had its own set of hidden data, separate from our training and test sets. It used that hidden data to evaluate our learners and compare our learners to all of the other teams. Our team, named "Fizzington Bisland the III," placed 28 out of 212 with a score of 0.76893. The number 1 team had a score of 0.78668, which is quite ahead. On the other hand, the last place team had a score of 0.49811, which meant that there was some error when training their model.
