About

     is a machine learning project for a course in data mining and machine learning at UCI (COMPSCI 178). The goal of this project was to place as high as possible in a class competition on Kaggle. The placement is dependent on the score your learner receives as it tries to predict data; the higher the score, the better.

     got a score of 0.78668, and placed 28 out of 212 teams. The first place team had a score of 0.76893, which I believe they managed through lots of optimization in learning parameters.

Usage

     has several different components: a neural network, a random forest, and an ensemble of the two. Each one is trained on rainfall location data, based on processed infrared satellite image information.

    To use , you simply run any of the three scripts pertaining to the three learners. Make sure you have data of some kind available for it to train on. You can then give it separate test data, and it will attempt to predict based on its model. It will be given scores based on how well it predicted the data.

Development

Overview

     was made in Python 2.7. A basic, generic learner was provided by the teaching staff. We utilized many different python libaries to train and evaluate our models: keras, scikit-learn, numpy, matplotlib, and collections.

Neural Network

    The neural network was created and trained with keras, and evaluated with scikit-learn. Neural networks are a series of layers, each of which consists of nodes. At the end of each layer is an activation function that transforms the data of that layer for the next layer. Essentially, a neural network is a series of many linear equations, where each set of linear equations transforms their outputs as weights for the next set. There are three types of layers: input, hidden, and output. The input layer begins initialized at random, all 0's, or by some other initializer. The output layer creates a single, or more outputs. The hidden layers transform the data from the input to the output.

    Our neural network began with a single hidden layer. I tried to make the neural network simple- having only a single hidden layer, and a few nodes. The network ended up performing extremely poorly, around 0.5 to 0.6. I then made it much more complex, adding more layers and many more nodes- around 50 nodes. The neural network performed much better, but still poorly relative to the other learners. We initially attributed this poor performance to overfitting (a learner being too complex so that it cannot make general predictions accurately), but later found that the network was actually underfitting (a learner being too simple so that it cannot make any accurate predictions). I increased the number of hidden layers and nodes, and added some optimizations, such as initializing the input layer with uniformly random numbers, setting the activation function to be the rectifier function for the hidden layers and the sigmoid function for the final layer, and setting the neural network to minimize the binary cross-entropy loss. It ended up performing about as well as the other learners.

Random Forest

    Our random forest was created from the base learner. A random forest is simply a group of decision trees. Each tree is trained with a random subset of the data or features, and each tree makes its own prediction. The random forest then decides on the actual prediction by taking some kind of majority vote.

    For our random forest, we also gradient boosted every single decision tree. We lowered the number of features selected, and provided parameters for the maximum tree height allowed, minimum number of leaves for each tree, and the total number of trees in the forest. We constantly changed those parameters as we were training and testing the learner, to see which combination performed the best. In the end, these changes did not contribute too much to the overall score, and the random forest ended up performing much better than the neural network.

Ensemble

    The ensemble was built directly from the neural network and random forest scripts. Each learner was trained on the same data, and would make their own predictions. The ensemble would then choose the majority prediction.
