DnD Lookup

Jul 2017 - Current

Personal, Database, In Development

Github Link

What is DnD Lookup?

    DnD Lookup is a database utility program that lets you search up monsters, items, spells, classes, and races for Dungeons and Dragons 5e. It is made with user experience, and information clarity in mind. It presents all relevant information for the searched topic in boxes, with all related information grouped together. It shows any related searches, and lets you filter and sort the results.

Development

    The idea for this program came to me when playing Dungeons and Dragons, and finding it difficult to flip through the books and pdfs (even with the built-in pdf search) to find a particular spell or monster. I wanted the user interface of the program to be the most important aspect, so I decided on building it in Unity3D and C#.

    The program stores monsters, items, spells, and other data in an sqlite3 database, using SQLiteStudio, and later DataGrip. In Unity, I created scripts to help insert data into the database (much easier than inserting in SQLiteStudio or DataGrip).

    The user interface was made entirely with Unity assets, and Dalelands font. The user interface consists of (1) a search bar, (2) filters, (3) results list, and (4) data display.

The interface. You can see all of the major parts of the program.

    The search bar also includes a drop-down menu which lets you choose what type of data you are searching for (i.e. monster). Entering nothing will list all data of that type.

The search bar. You can select between a variety of data types.

    The filters change depending on the type of data. For example, monsters can be filtered by type, size, or by their challenge rating range. On the other hand, items would be filtered by type, rarity, and value. You can select one or more filters, or you can select the all filter.

The filters. Selecting all while select all filters of that category, or de-select all.

    The results list is two-part: a sort by section, and the list. The sort by lets you choose how the results are listed. Like the filters, the sorting parameters depend on the data type. The results list will display all relevant data that matches your search parameters. Depending on the data type, additional, important information is listed along with their names (useful for quick glances). Selecting on one of the items in the list will fully display that item on the right-hand side.

The result list. Above the list is the sort by section. You can see at a quick glance all important information for each item in the list. For monsters, this would be the name, type, and challenge rating.

    The data display displays all information of the selected item in the result list, organized and grouped by category.

The display. You can see all information pertaining to the selected item. For mosnters, this would be the stats, features, and actions.

    The program was designed with the MVC model in mind. The models (database) holds all of the data needed to function, and the controllers query the database, storing returned data into customized data structures for the views to display to the user. When a user types a query into the search bar, the program performs select queries into the database through a database controller. Then the database controller returns a data structure holding the information to another controller. That controller updates the views for the user to see.

Current Status

    The program is nowhere near finished. At the moment, only the monster portion is implemented. However, the database retrieval and displays are fully implemented, and the only tasks required are data type-specific ones, like spell tables, spell data, spell filters, and the spell display layouts.

Planned Features

    The following are planned, and will eventually be implemented (You can skip this part if you have no interest in DnD 5e):

  • Backgrounds: their description, traits, and features.
  • Classes: their description, features, archetypes, and builds.
  • Conditions: their effects.
  • Features: their effects, classes, and requirements.
  • Items: their description, effects, and rarities.
  • Races: their description, traits, subraces, and builds.
  • Spells: their description, classes, types, and stats.

    Almost everything from the SRD is planned to be included, as well, as the ability to create your own backgrounds, classes, items, races, etc. I also need a more creative name. Any suggestions?
