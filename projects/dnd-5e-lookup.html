About

     is a database utility program made for searching through Dungeons and Dragons 5e content. This includes monsters, items, spells, and more. Users can search up whatever they are looking for, and will display all relevant information in a neat and organized display. Due to licensing, will only include SRD content. Note that the images shown here include non-SRD content. That content was used primarily for testing.

Typical usage of the . A user has searched for a monster with the Celestial, Dragon, or Fey types, and Large, Huge, or Gargantuan sizes with a challenge rating between 10 and 30.

The four parts of : (1) the search bar, (2) the search filters, (3) the results list, and (4) the display.

Usage

Search Bar

    You can use by running the program, and then entering whatever search query you wish to search for into the search bar. You can set the type of data you want to search in:

  • Background
  • Class
  • Condition
  • Feature
  • Item
  • Monster
  • Race
  • Spell

The search bar. You can select which data type to search in using the dropdown menu.

Search Filters

    Once set, you can also filter out certain subtypes. Different data types have different filters. For example, Monsters can be filtered by type, challenge rating, and size. Spells can be filtered by school, what classes can cast this, and level. Items can be filtered by type of item, weight, value, and rarity.

The search filters. You scan select or deselect any of these filters. Some filters are range-based, instead like the challenge rating. Finally, you can select all to select or deselect all filters in that category.

Sorting Metrics

    You can also sort how the results are displayed. Just like the filters, different data types have different sorting metrics. Monsters can be sorted by name, challenge rating, type, size, tags, and source material. Spells, on the other hand, can be sorted by name, school, class, level, and source material.

The results list. You can sort the results by various metrics. You can also set the sorting to be ascending or descending. You can see, at a quick glance, important information for each result within the list.

Results and Display

    Once the results come in from a search query, you can select any result to display it onto the display window. This will display all relevant information in an organized way. Monsters show their statistics, features, and actions. Spells show their statistics, effect description, and classes. Items will show their statistics, and description.

The display. Here, you can see all relevant information, grouped up by function. For the monster here, you can see the statistics grouped up in one area, and the features and actions in another.

Real-Time Search Modifications

    Modifying the search query, or changing the filters when results are already shown will update the results in real-time.

The results list automatically updates whenever you change the filters or sort metrics.

Development

Overview

     is made in Unity3D and C#. The database is made in SQLite3 via SQLite Studio and DataGrip. However, the actual insertion of data into the database is done via C# scripts that send INSERT queries to the database. The user interface is made entirely with Unity assets, and the Dalelands font. is made with the MVC pattern in mind. The database acts as the data layer, while the several managers act as the controllers. Finally, the UI acts as the view. Custom data structures are also used to transport data between the three.

Database Management

    The database is managed by two database controller scripts made in C#. The first, called the handler, handles creating SELECT queries from the search parameters, and retrieves data from the database. The handler stores this data into custom data structures, which are then sent back. The second, called the loader, handles creating INSERT queries out of data from the Unity inspector. The insertion of data into the database is done via a custom C# script that lets me enter data through the inspector (this is much easier than manually entering the data in SQLite Studio, or manually making INSERT queries in DataGrip).

User Interface

    The user interface is managed by a view controller called UI manager, and many separate behaviour scripts. The search bar, search filters, and results list views provide search parameter data to the view controller. The UI manager will then send that data over to the handler, and return with data structures containing search result data. Next, the UI manager will send certain data to the results list view for it to display. Finally, when you select one of the results in the results list, the UI manager will send the appropriate data structure to the display view.

Individual UI Components

    The UI manager is aided in its task by several UI components. Each of these have a small, but very specific job, and they handle specific parts of the UI. For example, the filter control handles the search filters, and communicate directly with the UI mananger when filters are changed. It simply checks whether each of its values are toggled or not, and sets flags when a change is made. For the non-select filters, the range control handles them. Similar to the filter control, this simply checks whether the values in their fields have been modified.

Future

Current Status

     is currently in development. This was made as a learning experience in C#, SQLite3, Unity3D, data management, and UI design. Since it is early in development, it only has a few features:

  • Different data types: Ability to switch data types when searching.
  • Data-dependent filters: Filters available that change depending on the data type.
  • Data-dependent sort metrics: Sort metrics available that change depending on the data type. Can choose ascending or descending.
  • Real-time updates on results: Results are updated in real-time based on filter and sort changes.
  • Display: Displays all relevant information in an organized fashion for the selected result.
  • In-inspector insertions: Insert into the database through the Unity inspector.

Planned Features

    I have many features and ideas planned for . The most important one is refactoring and re-organizing the code. As this was meant to be a learning experience, the code is messy and unoptimized. This leads to difficulty when trying to add new data types, adding new data into the database, and creating the filters and sort metrics. The important features planned are:

  • All SRD content: Include all SRD content.
  • More data types: Include backgrounds, classes, conditions, features, items, races, spells, and more.
  • Caching: Improve performance by caching query results. Change the cache for the real-time updates, instead of making new queries.
  • Better UI: Improve UI elements and layout so that it is clearer and sharper.
  • Custom Data: Include ability to make your content.
  • Character Creator: Include ability to make a character.
