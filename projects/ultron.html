About

     is an English language processor made for a data structures course at PCC (CS 8). accepts English sentences from the user, and answers any questions about what it has been told. This project is a very simple application of the very complex topic of natural language processing.

Usage

     requires English sentences as inputs. In the beginning, it knows nothing. These inputs provide a knowledge base for . These sentences must follow the provided grammar rules. Currently, only one such set of rules are allowed. This leads to extremely simple sentences with transitive verbs, like "John kicked the ball."

    You can also ask simple questions about what it knows- questions like, "What did John kick?" will look for an answer in its knowledge base, and give you an answer.

User types "John throws the ball." The user then asks "What did John throw?" responds with "ball."

Development

Overview

     was made in C++ via Qt Creator. When the user inputs a sentence or question, tokenizes the input, tags each individual word with a domain of parts of speech, then parses it into syntax trees, and finally stores it into its knowledge base or queries its knowledge base for an answer.

Tokenizing and Tagging

     first tokenizes the input string into individual tokens or words. Then, for each word, it queries its ontological database for the word and its possible parts of speech. It tags that as the token's domain. It then sends the tokens and their domains to a parser to determine which assignments (token + part of speech) is the most likely.

Parsing

     parses the tokens using recursive descent parsing. Essentially, it follows a set of context-free grammar rules. The parser begins at the first token, and checks each part of speech in its domain against the first grammar rule. If there is not match, it moves onto the next set of grammar rules. If there is a match, it creates a syntax tree with the grammar rule as its root. Then, it moves onto the next token, and the next rule that follows from the matching grammar rule. It continuously builds up the syntax tree, until every token has a matching and legal part of speech. If at any point, it cannot continue building the tree, it backtracks until there are no more options./p>

An example context-free grammar and syntax tree. Courtesy of http://www.cs.uccs.edu/~jkalita/work/cs589/2010/12Grammars.pdf

Database Management

     has two databases: one for the English words, and one for the knowledge base. Both were made using SQLite. The ontological database stores the most common words and their parts of speech. uses this to determine what parts of speech each word in an input sentence is. The second database stores words provided to it by the user. This is used when is asked a question.

Future

     is a completed project for a course. We were hoping to use predicate calculus and AI searching algorithms so that it can extrapolate logically from input sentences, and also to use probability to more accurately determine the parts of speech. However, when researching about this topic, we realized we were in way over our heads. We decided to do the best we could, and ended up with this simple processor.
