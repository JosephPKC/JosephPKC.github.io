About

     is a database program to help users get information about classical Greek vocabulary. You can enter vocabulary words in Greek, and the program will search and display any and all information on that word. This information includes data like definitions and forms.

Usage

Commands and Arguments

     is a command-line program, and supports a variety of commands:

  • info, i: Get information on a given Greek word.
  • list, li, l: List all Greek words of a given type.
  • update, up, u: Update the database.
  • help, h, ?: List the commands and their uses.
  • quit, q: Exit the program.

    The info command also supports a few optional arguments:

  • -verbose, -v: Display the entire form of the given word, instead of just a gist.
  • -define, -def, -d: Display the definitions of the given word as well.
  • -unaccent, -u: Ignore accent and breathing marks.

The info command, with the arguments of -v, -d, -u. has displayed the full form info and definitions of the Greek word, 'παιδεύω.' Notice how the second command has no arguments; thus, the result is not as detailed.

Development

Overview

     is made with Python 2.7 via Pycharm. The database is made in SQLite3, through SQLiteStudio. You type in a command, and any arguments (usually, Greek words), and then will query the database, retrieve appropriate information, and display it.

Processing

     parses and accepts user input through a controller. This controller determines what command the user is looking to execute, gets any arguments, and executes that command. The controller also communicates with the database handler when it needs information. It also retrieves any information from the handler, and displays it to console.

Database Management

     stores information on Greek words in an SQLite3 database. The database handler communicates with the database, and retrieves the requested information from it. It is given a Greek word, and searches the general table for information on its part of speech. With the part of speech, the handler searches the appropriate table for more specific information. Depending on the arguments, the handler will then create a data structure to hold all of the information and return it.

Data Loading

     loads new Greek words and information through the same database handler. When the update command is executed, the controller communicates with a text parser that opens up the lexicon text file, parses it, and returns it. The controller then communicates with the database handler to send INSERT queries to the database.

This is a part of the lexicon text file. Through this file, you can insert new words into the database without having to manually create queries. This is specically for the Greek word 'παιδεύω' meaning 'teach.'

Future

Current Status

     is currently in development. This was made for use in my classical Greek courses at UCI. However, since I have graduated, I am no longer in need of this program. I am still planning on developing this, but it does not have a high priority.

    Currently, has these features implemented:

  • Info queries: You can search for information about many Greek words.
  • Additions via text: You can insert new words via text file.
  • Unaccented forms: All Greek words can be searched for unaccented.

Planned Features

    There are many things that can be done to improve . This includes:

  • Optimizations: Caching, and other optimizations can improve performance.
  • GUI-based insertions: GUI tool to insert and modify words in the database without the use of text files.
  • Irregular words: Addition of words that do not follow the grammar rules.
  • Word stem and ending predictors: Heuristic algorithms to try and predict word stems and word endings, based on the grammar rules.
  • Sentence translations: Translation of simple Greek sentences, using searching alrogithms.
