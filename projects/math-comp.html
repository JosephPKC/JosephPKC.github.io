About

     is a Hacktech 2017 project. We utilize various APIs in order to calculate mathematical equations written on a touchpad: the Synaptics Touchpad API to read in sensor data from the touchpad, the Google Cloud Vision API for image recognition to determine what was written onto the touchpad, and the Wolfram Alpha API to calculate the mathematical equation.

Usage

    To use , you run the image script. Connect a Synaptics touchpad, and begin writing whatever equation onto it. Once you stop writing, the program will capture the writing as an image and display it.

Writing out 7 + 7 into the touchpad. It recognizes the equation correctly, and gives the correct answer.

Writing out a smiley face. Unfortunately, it did not recognize it as a smiley face.

Writing out sin 7/2. At least it got the sine part right.

    Once the image is saved, run the main script, and it will send the image through Google Clouda Vision's learner, and spit out a math equation string. The string is then taken through Wolfram Alpha, and an answer is displayed.

Development

Overview

     was made in Python 2.7. There were several components to this project: the touchpad interface, the image recognition, the calculation, and the user interface.

Touchpad Input

    The input from the Synaptics touchpad was given as a 2d array. The code for this was given to us by Synaptics with the touchpad, so all we had to do was create an interface for the code. We eventually made the interface grab the 2d array, and create an image file of it and save it.

Image Recognition

    The creation of the image was due to how the image recognition worked. It required an image, so that it can try and predict what it was saying. Once the image was saved, it was passed into the Google Cloud Vision interface. It tried to figure what the equation was, and sent it back as a string.

Calculation

    After the string is given by Google Cloud Vision, the interface sends that string to Wolfram Alpha. It then calculates it and sends back a dictionary of answers- all sorts of different answers. This could be just the direct answer, a graph showing how the equation grew, or even limits. We decided that we just wanted to most straightforward answer, and chose specific ones for different types of equations- the direct answer for simple equations, variable assignments for multi-variable equations, etc.

User Interface

    All of this was displayed to the user via a user interface created in pygame. The user interface showed the image that was captured from the touchpad, what Google Cloud Vision thought the image was saying, and what Wolfram Alpha thought the answer was.

Future

     is a completed project. At Hacktech, we were trying to figure out the best way to incorporate as many APIs as possible. The best we could think of in a short amount of time was a calculator that took input in the form of hand-writing on a touchpad.

    There are many improvements that can be made for. The most glaring issue we found was that it just simply did not recognize equations well. We later realized that at the time, the image recognition API we were using was not optimized for numbers or equations. Thus, a big improvement would be to either use a different image recognition API that is more optimized for numbers, or train our own model. Another big improvement would be fitting the entire program into a single executable. Currently, you must run two diffferent ones- one for the image capture, and one for the calculation. Finally, the user interface could use a lot of work.

    Despite all of the shortcomings of , I am still proud of the work my team and I had done keeping in mind that we only had 3 days to learn the APIs and create this project. While we didn't win any prizes (as there were many projects that were very good and much better than ours), I believe we still made something we can be proud of.
